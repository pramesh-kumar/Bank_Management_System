🧱 Account Struct — The Data Box

struct Account
🪪stringid
👤stringname
👨stringfname
🏠stringaddress
🔢stringpin
🔒stringpass
📱stringphone
💰floatbalance

A struct is just a box that groups related data together.

Instead of having 8 separate variables floating around, you put them all in one Account object.

When you pass an account to a function, you pass one thing — not 8.

In bank.txt, each line = one Account:

U001 Alice Bob Mumbai 12345 pass1 9876543210 5000.00
U002 Dave Smith Delhi 99887 pass2 9123456789 1200.50

💾 How rewriteDB() Works — Update & Delete

You can't edit one line in the middle of a text file. So we use the temp-file-swap trick:

📝 UPDATE (e.g. deposit to U002)

bank.txt (before)
U001 Alice ... 5000
U002 Dave ... 1200
U003 Sara ... 3000
original file
→
temp.txt
U001 Alice ... 5000
U002 Dave ... 1700 ✓
U003 Sara ... 3000
temp file
→
bank.txt (after)
U001 Alice ... 5000
U002 Dave ... 1700
U003 Sara ... 3000
renamed from temp

🗑️ DELETE (e.g. delete U002)

bank.txt (before)
U001 Alice ... 5000
U002 Dave ... 1200
U003 Sara ... 3000
original file
→
temp.txt
U001 Alice ... 5000
U002 Dave ... 1200
U003 Sara ... 3000
U002 skipped
→
bank.txt (after)
U001 Alice ... 5000
U003 Sara ... 3000
renamed from temp

⚙️ What Each Function Does — Step by Step

createAccount()
1. Ask user for all fields
2. Call findAccount() to check duplicate
3. If no duplicate → write to bank.txt
deposit()
1. Ask for ID and amount
2. Check amount > 0
3. findAccount() → get current balance
4. balance += amount
5. rewriteDB() → save updated account
withdraw()
1. Ask for ID and amount
2. Check amount > 0
3. findAccount() → check balance
4. If amount > balance → STOP ✗
5. balance -= amount → rewriteDB()
transfer()
1. Ask sender ID, receiver ID, amount
2. findAccount() for both
3. Check sender has enough balance
4. sender.balance -= amount
5. receiver.balance += amount
6. rewriteDB() twice (one per account)
payBill()
1. Ask ID, bill name, amount
2. findAccount() → check balance
3. balance -= amount → rewriteDB()
4. Log entry into bill.txt
searchAccount()
1. Ask for ID
2. findAccount() → get data
3. Print details
4. ⚠️ PIN/password NOT printed
editAccount()
1. Ask for ID → findAccount()
2. Ask for new values
3. rewriteDB() with updated account
deleteAccount()
1. Ask for ID → findAccount()
2. rewriteDB(id, nullptr)
3. nullptr = skip that record = deleted

🏧 ATM — authenticate() Flow

authenticate() — called before every ATM action
1
Ask: User ID, PIN, Password
2
findAccount(id) → does it exist?
No → "Account not found" → STOP
3
Does pin match AND password match?
No → "Invalid PIN or Password" → STOP
4
✓ Return true → proceed to action

Why is ATM separate from Bank Management?

Bank Management = admin panel (no login needed)

ATM = customer self-service (needs 3-factor check)


The authenticate() function is reused by both:

• atmCheckBalance()

• atmWithdraw()


This avoids writing the same login check twice — DRY principle (Don't Repeat Yourself)

🔍 Full Trace — What happens when user deposits money

main() — user picks "1. Bank Management"
↓
bankMenu() — user picks "3. Deposit"
↓
deposit() — asks "User ID:" → U001   "Amount:" → 500
↓ check: amount > 0 ✓
findAccount("U001") — opens bank.txt, reads line by line, finds U001, copies into Account a
↓ a.balance = 5000
deposit() — a.balance += 500 → now a.balance = 5500
↓
rewriteDB("U001", &a) — reads all accounts, writes them to temp.txt, replaces U001 line with new balance, renames temp.txt → bank.txt
↓
bank.txt updated — U001 now shows balance 5500 ✓
↓
bankMenu() — prints "Deposited. New balance: 5500" → loops back to show menu again
